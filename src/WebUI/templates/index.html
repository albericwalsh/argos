Variables Jinja2

Flask passe des variables Python au template via render_template(). Dans le HTML, on les affiche avec la syntaxe {{ '{{' }} variable {{ '}}' }}.

app.py 
return render_template('index.html',
    APP_NAME    = APP_NAME,
    APP_VERSION = APP_VERSION,
    WEB_SERVER_HOST = WEB_SERVER_HOST,
    WEB_SERVER_PORT = WEB_SERVER_PORT,
)
index.html 
<h1>{{ '{{' }} APP_NAME {{ '}}' }}</h1>
<p>Version : {{ '{{' }} APP_VERSION {{ '}}' }}</p>
Valeur injectée en live : cette page affiche {{ APP_NAME }} v{{ APP_VERSION }} — ces valeurs viennent de Python.

Conditions & boucles

Jinja2 supporte {{ '{% ' }}if{{ ' %}' }} et {{ '{% ' }}for{{ ' %}' }} pour du HTML dynamique.

Condition 
{{ '{#' }} Afficher un bloc selon une variable {{ '#}' }}
{{ '{% ' }}if APP_VERSION == '1.0.0' {{ '%}' }}
  <p>Première release !</p>
{{ '{% ' }}else{{ ' %}' }}
  <p>Version {{ '{{' }} APP_VERSION {{ '}}' }}</p>
{{ '{% ' }}endif{{ ' %}' }}
Boucle — app.py passe une liste 
# app.py
return render_template('index.html', items=['a', 'b', 'c'])

{{ '{#' }} index.html {{ '#}' }}
<ul>
{{ '{% ' }}for item in items {{ '%}' }}
  <li>{{ '{{' }} item {{ '}}' }}</li>
{{ '{% ' }}endfor{{ ' %}' }}
</ul>

Fichiers statiques (CSS / JS / images)

Flask sert automatiquement le dossier static/. Utilise url_for pour construire les URLs.

Structure 
project/
├── app.py
├── templates/
│   └── index.html
└── static/
    ├── style.css
    ├── app.js
    └── logo.png
index.html — lier les assets 
<link rel="stylesheet" href="{{ '{{' }} url_for('static', filename='style.css') {{ '}}' }}">
<script src="{{ '{{' }} url_for('static', filename='app.js') {{ '}}' }}"></script>
Pourquoi url_for ? Il génère le bon chemin même si l'app est montée dans un sous-répertoire — ne jamais écrire /static/... en dur.

Appels API depuis le front (Fetch)

Pour rendre la page interactive sans recharger, utilise fetch() en JavaScript pour appeler tes routes Flask.

app.py — route JSON 
from flask import jsonify

@app.route('/api/status')
def status():
    return jsonify({ 'ok': True, 'app': APP_NAME })
static/app.js — appel fetch 
async function getStatus() {
  const res  = await fetch('/api/status')
  const data = await res.json()
  console.log(data.app)  // → "MyApp"
}

getStatus()

CORS — Cross-Origin Resource Sharing

Le CORS permet à un front servi sur un autre domaine/port (ex : React sur :3000) d'appeler ton API Flask (:8080). Sans CORS, le navigateur bloque la requête.

Header HTTP Rôle Valeur par défaut ici
Access-Control-Allow-Origin Domaines autorisés localhost::8080
Access-Control-Allow-Methods Méthodes HTTP permises GET, POST, PUT, DELETE, OPTIONS
Access-Control-Allow-Headers Headers acceptés Content-Type, Authorization
variables.py 
CORS_ORIGINS = ["http://localhost:8080"]
CORS_METHODS = ["GET", "POST", "PUT", "DELETE", "OPTIONS"]
CORS_HEADERS = ["Content-Type", "Authorization"]
En production : remplace localhost par ton vrai domaine. N'utilise jamais "*" si tu envoies des credentials (cookies, Authorization header).
